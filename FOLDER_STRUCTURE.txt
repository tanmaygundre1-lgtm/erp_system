KEY PATTERNS AND CONVENTIONS:

1. FEATURE MODULES ARE SELF-CONTAINED
   Each feature (auth, users, products, orders) is a vertical slice containing
   all layers needed for that domain.

2. INDEX FILES FOR CLEAN IMPORTS
   Each folder contains index.ts/index.tsx for barrel exports:
   тЬЕ import { useAuth } from './hooks'
   тЭМ import { useAuth } from './hooks/useAuth'

3. NAMING CONVENTIONS
   - Files: camelCase.ts (authService.ts, userController.ts)
   - Components: PascalCase.tsx (UserCard.tsx, LoginForm.tsx)
   - Folders: lowercase (components/, services/, hooks/)
   - Types: PascalCase.ts (User.ts, Product.ts)

4. STRICT LAYER SEPARATION
   тЬЕ Components тЖТ Services тЖТ Hooks (circular)
   тЬЕ Services тЖТ Models тЖТ Database
   тЭМ Components directly calling database
   тЭМ Models having business logic

5. SHARED VS FEATURE-SPECIFIC
   - Shared: Reusable across multiple features (Button, Modal, useLocalStorage)
   - Feature-specific: Only used within one feature (LoginForm, useAuth)

6. API CALLS
   Frontend: service/ тЖТ HTTP calls тЖТ Backend: controller/ тЖТ service/
   Backend: controller/ тЖТ service/ тЖТ model/ тЖТ database/
